Part 1: Deployment & Updates

‚ö†Ô∏è THE "ONE-SHOT" RULE (Critical)

The App (Frontend) and the Database (Backend) share a secret "Worker Key". This key is regenerated every time you refresh the Factory page.

To update the system, you must do steps 1-3 in ONE session without closing the tab:

  1. Copy Script: Paste the code into your Google Sheet Script Editor and Save.
  2. Download ZIP: Download the App files immediately after.
  3. Upload App: Put the new App on Netlify.

If you refresh the Factory page in between, the keys will not match, and you will get an "Access Denied" error.

1 The Database (Google Sheets)
  1. Go to sheets.new. Name it "OTG Safety DB".
  2. Create exactly 4 tabs: Visits, Staff, Templates, Sites.
  3. Copy Headers: Use the "Copy Headers" buttons in the Factory App (Step 2) to set these up correctly.
2 The Backend (Apps Script)
  1. In your Sheet, go to Extensions > Apps Script.
  2. Open the file Code.gs included in this ZIP. Copy all text.
  3. Paste it into the Script Editor (replace default code). Save.
  4. Deploy: Click Deploy (Blue Button) -> New Deployment.
  5. Select Type: Web App.
  6. Configuration:
    • Execute as: Me.
    • Who has access: Anyone (Critical for App connection).
  7. Click Deploy -> Authorize Access -> Copy the Web App URL.
3 Hosting the Apps

Use a static host like Netlify Drop (free).

  1. Sign up at Netlify.com.
  2. Drag the WorkerApp folder from this ZIP onto the Netlify dashboard.
  3. Once uploaded, click "Sign up to claim site" so the link doesn't expire in 1 hour.
  4. Repeat this process for the MonitorApp folder.
