Part 1: Initial Deployment

1 The Database (Google Sheets)
  1. Go to sheets.new. Name it "OTG Safety DB".
  2. Create exactly 4 tabs: Visits, Staff, Templates, Sites.
  3. Use the "Copy Headers" buttons in the Factory App to set these up.
2 The Backend (Apps Script)
  1. In your Sheet, go to Extensions > Apps Script.
  2. Paste the Code.gs content from the Factory.
  3. Deploy: Click Deploy (Blue Button) -> New Deployment.
  4. Select Type: Web App.
  5. Configuration: Execute as: Me, Who has access: Anyone.
  6. Copy the Web App URL.
3 Hosting the Apps

Use a static host like Netlify Drop (free).

  1. Drag the WorkerApp folder to Netlify.
  2. Important: Click "Sign up to claim site" so the link doesn't expire.
  3. Repeat for the MonitorApp.

Part 4: Updates & Modifications

‚ö†Ô∏è THE "NEW VERSION" TRAP

When you edit code in Google Apps Script, clicking "Save" DOES NOT update the live system.

For any change to take effect, you must:

  1. Click Deploy (Blue Button).
  2. Select Manage Deployments.
  3. Click the Pencil Icon (Edit).
  4. Change "Version" to "New version".
  5. Click Deploy.

Scenario A: Performing a "Soft Update" (Code Only)

Use this method to update the Worker App features without forcing everyone to reinstall.

  1. Open the Factory App (index.html).
  2. CRITICAL: Paste your saved Master Key and Worker Key into the configuration boxes.
  3. Click Generate Backend Code.
  4. Copy the new code and paste it into your Google Apps Script editor.
  5. Perform the "New Version" deployment steps above.
  6. Download the new AppSuite ZIP.
  7. Upload the new WorkerApp folder to Netlify (drag and drop it onto your existing site dashboard to update it).
  8. Tell staff to Close and Reopen their app twice to pull the update.

Scenario B: Modifying the Backend Logic

You can edit Code.gs directly to change how the system behaves (e.g., AI prompts, Email formatting).

  1. Open your Google Sheet > Extensions > Apps Script.
  2. Find the function you want to change (e.g., smartScribe for AI, sendAlertEmail for notifications).
  3. Make your changes.
  4. Test: Click the "Run" button to test specific functions (like checkOverdueWorkers).
  5. Deploy: Follow the "New Version" deployment steps above.
