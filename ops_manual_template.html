Part 1: Deployment Guide

Follow these steps to bring your serverless system online.

1 The Database (Google Sheets)
  1. Go to sheets.new. Name it "OTG Safety DB".
  2. Create exactly 4 tabs: Visits, Staff, Templates, Sites.
  3. Copy Headers: Open this file in a text editor to see the column headers required, or use the buttons in the Factory App to copy them.
2 The Backend (Apps Script)
  1. In your Sheet, go to Extensions > Apps Script.
  2. Open the file Code.gs included in this ZIP. Copy all text.
  3. Paste it into the Script Editor (replace default code). Save.
  4. Deploy: Click Deploy (Blue Button) -> New Deployment.
  5. Select Type: Web App.
  6. Configuration:
    • Execute as: Me.
    • Who has access: Anyone (Critical for App connection).
  7. Click Deploy -> Authorize Access -> Copy the Web App URL.
3 Hosting the Apps

Use a static host like Netlify, Vercel, or GitHub Pages.

  1. Sign up at Netlify.com.
  2. Drag the WorkerApp folder from this ZIP onto the Netlify dashboard.
  3. Once uploaded, go to Site Settings > Change Site Name (e.g. acme-safety.netlify.app).
  4. Repeat this process for the MonitorApp folder.

3. Managing Alerts

When an alarm triggers (Red/Purple):

  1. Acknowledge: Click the big "Acknowledge" button to silence the siren locally. This DOES NOT clear the alarm on the server.
  2. Resolve: Once you have contacted the worker and confirmed safety:
    • Click the "âœ… RESOLVE ALERT" button on their card.
    • Enter a Note (e.g., "False alarm, spoke to John").
    • Click Confirm. This sends a "Green Alert" (All Clear) email to the emergency contacts.

4. Map View & Breadcrumbs

Switch to Map View using the toggle in the top right.

  • Live Location: Shows the last known GPS ping.
  • Breadcrumbs: A dotted blue line shows the worker's path (last 10 pings).
  • GPS Quality: If the pin is missing, it means the phone hasn't sent valid GPS data yet.

Part 3: The AI Co-Pilot

This system provides you with the raw source code (`.html` and `.js` files). You can use AI tools (Gemini, ChatGPT) to customize it without knowing how to code.

How to Prompt for Changes

To get working code, you must provide the AI with context. Always paste your current code first.

Scenario A: Changing Colors

"Here is the CSS from my worker_template.html file: [PASTE CODE CHUNK] Please change the primary branding color from Blue (#3b82f6) to Safety Orange (#f97316). Keep the functionality the same."

Scenario B: Adding a Field to the Sign-On Wizard

"I have a Setup Wizard in my worker_template.html. [PASTE WIZARD HTML CODE] Please add a new input field for 'Employee ID Number'. It should be mandatory. Save it to the settings object so it is included in every report."

Scenario C: Creating a Custom Report Script

"Here is the structure of my Google Sheet 'Visits' tab: Col A: Date Col C: Worker Name Col S: Distance Please write a Google Apps Script function called 'emailMonthlyMileage'. It should sum the distance for each worker for the current month and email me a summary table."

Best Practices

  • Backup First: Always keep a copy of your working file before pasting AI code.
  • Be Specific: Describe what you want, not how to do it.
  • Iterate: If the code errors, copy the error message and paste it back to the AI.
